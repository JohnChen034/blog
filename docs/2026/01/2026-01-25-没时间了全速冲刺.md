# notes on papers

## [AI-MARRVEL](https://pmc-ncbi-nlm-nih-gov.offcampus.lib.washington.edu/articles/PMC11221788/)


Q: Why these kind of tools?

Every people's exome has tens of thousands of variants, and then after common bioinfo tools for common/low quality variants, we still left with hundreds. Manually review them is very time consuming. So we need more sophisticated methods to increase our speed of diagnose. 

Moreover, we discover novel disease genes every year, so periodic reanalysis on unsolved cases can be useful. therefore need more efficient tools. 

So variant filer+prioritization tools comes, like VAAST,14 Phevor,15 Phen-Gen,16 PhenIX,17 Exomiser,15 Phenolyzer,18 Genomiser,19 Xrare,20 LIRICAL,21 AMELIE,22 GEM,23 MOON,24 Emedgene... However, they are limited in noncoding variant, using simulated data, and lack of acuraracy. 


AI-MARRVEL uses clinical features and sequencing profiles. 



## HOW

**Knowledge-Based Feature Engineering:**

 raw features, patient variants were annotated using:
 
 - VEP 
 - additional databases summarized by marrvel.org31 (including DGV: the Database of Genomic Variants,36 DECIPHER: Database of Chromosomal Imbalance and Phenotype in Humans using Ensembl Resources,37 ClinVar,38 and OMIM: Online Mendelian Inheritance in Man39).

input data size =  {n_s * n_v * (P+1)}

validate using some simple topk weighted equation. 

To validate, for every validation sample, first form candiate variant list, then calculate all features, then output scores for every vairant, then order top1/5/10.
